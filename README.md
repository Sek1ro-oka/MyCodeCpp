# 用户类：
必做：姓名（string），ID（string），密码（string），性别（string），手机（string），电子邮箱（string），星级（double），信誉分（double），发布委托（vector<string>），接受委托（vector<string>）根据信誉分计算星级

选做：封禁标志（bool），用户不可登录该账号，须向管理员申请恢复，收入（可提现）

写入格式：姓名，ID，密码，性别，手机，电子邮箱，星级，信誉分，发布委托数量，依次写入发布的委托的编号，接受委托的数量，依次写入接受委托的编号
中间用空格分隔，最后加换行，表示一个用户


委托类：
必做：时间（time_t），地点（string），内容（string），星级（double），报酬（double），星级由用户决定还是系统决定
取消标志（bool），接受标志（bool），完成标志委托者（bool），完成标志接受者（bool），接受者完成委托后须向委托者确认，两者同时确认才认定完成委托，否则，如有违约，委托者可进行申诉，取消委托
委托者姓名（string），ID（string），接受者姓名（string），ID（string），显示姓名，根据ID查找用户
申诉标志（bool），若双方有争议未解决（如未完成委托），委托者可向管理员申诉，申诉成功扣除信誉分
评分（double），评语（string），只有完成后才能评价，只能评价一次
删除标志委托者（bool），删除标志接受者（bool），如果有该标志，则指定用户不可见，而服务端可见

选做：审核标志（bool），审核反馈（string）
聊天记录编号（string）
委托人和被委托人的聊天记录，可以单独建一个文件夹存聊天记录，通过唯一的编号打开对应聊天记录，可查看被委托人信息
举报标志（bool），被举报后管理员再次审核，如不通过则删除该委托，并降低信誉分

写入格式：时间，地点，内容，星级，报酬，取消标志，完成标志，评分，评语……
